# Hybrid Cloud for AI

Hybrid cloud strategy is key to supporting growing adoption of AI and ML solutions. Enterprises use data as the main asset for empirical decisions, dealing with high volume, variation, and velocity.

## Why Hybrid Cloud?
- **Elastic resources**: Cloud provides on-demand compute and storage with pay-as-you-go pricing
- **Private data**: Sensitive customer data stays on-premises close to applications
- **Data preprocessing**: Analytics can anonymize and remove PII before uploading to cloud
- **Cloud storage**: Used for large datasets required for training ML models, taking advantage of scalability and performance
- **Model training**: Runs as batch processing (minutes to days) then releases resources — less relevant to buy expensive CAPEX hardware when cloud computing is available
- **LLM access**: Most Generative AI and LLMs are deployed as SaaS with API access

## Data Sources
Enterprise data about customer behaviors comes from:
- Click stream
- Shopping cart content
- Transaction history
- Historical analytics
- IoT sensors
